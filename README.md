# G-Browser (G浏览器) - 极简多功能个性化浏览器与音乐播放器

G-Browser 是一款基于 Android Jetpack Compose 开发的极简、轻量、高可定制性的移动端多功能应用。它不仅提供了高效流畅的多标签网页浏览、严密的隐私保护，还深度融合了实时定位天气问候、动态视频壁纸主页、后台音乐播放与双语歌词同步、自研并发下载管理器以及高级安全隐私配置，完美结合了实用浏览与多媒体娱乐体验。

---

## 🌟 核心功能详解

### 1. 🌐 智能双模浏览器
* **多标签页卡片式管理**：提供类似于系统卡片的标签预览（Tab Overview）界面，支持新建标签页、一键关闭、左右滑动删除标签。各标签页相互隔离 WebView 实例与状态，保证多网页浏览的独立性。
* **快捷引擎切换**：内置 **Baidu、Bing、Google、Sogou** 四大主流搜索引擎，点击输入框左侧图标可实现一键快速切换。
* **极速输入联想建议**：基于百度搜索建议接口，内置 **150ms 智能防抖（Debounce）过滤** 算法，有效减少不必要的网络请求，实现高响应度的智能词条联想。
* **智能网页夜间模式**：基于 WebView 系统的 ForceDark 硬件加速反色技术，自动反转网页中的刺眼浅色背景，同时智能识别网页内的图片与视频元素，保持其色彩不失真。
* **电脑版网页排版切换**：支持一键切换 User-Agent，快速在手机版和电脑版（Desktop Mode）网页排版之间平滑切换。
* **书签与历史记录**：支持对喜爱的网页一键加入书签，提供详细的历史记录浏览列表，支持单条删除和一键彻底清空。

### 2. 🌤️ 地理定位与实时天气问候
* **高精度 GPS 定位**：集成系统 `fusedLocationClient` 定位服务，在获取用户位置授权后，自动读取当前高精度经纬度。
* **三层容灾天气获取机制**：
  1. **首选 GPS 实时天气**：基于当前经纬度调用心知天气 API (Seniverse API) 获取当前的温度与天气状况。
  2. **第二层降级（默认城市）**：定位失败或网络超时，自动尝试获取默认城市（北京）的实时天气。
  3. **第三层降级（安全默认值）**：若全部接口故障，自动加载安全预设天气值（北京，22°C，晴），确保主页天气组件在任何恶劣环境下均不发生闪退。
* **动态天气 Emoji 适配**：根据获取到的天气状态描述（如晴、云、阴、雨、雪、雷、雾、扬沙等），自动匹配显示相对应的灵动 Emoji 图标（如 ☀️, ☁️, 🌧️, ❄️, ⚡ 等）。
* **时间段个性问候**：根据当前系统时间自动匹配不同时段的贴心问候文本（如“早上好”、“中午好”、“下午好”、“晚上好”、“夜深了”）。

### 3. 🎬 个性化动态壁纸背景
* **超清媒体壁纸库**：内置 15 张超清静态艺术壁纸与 10 个高清循环视频壁纸，支持在设置与主页间自由切换，让主页背景鲜活灵动。
* **本地相册自定义导入**：支持调用系统相册选择器，允许用户导入手机本地的任意图片作为主页的静态壁纸背景。
* **GPU 渲染与能耗智能优化**：
  * **后台自动挂起**：当 App 退到系统后台时，视频壁纸解码器 (ExoPlayer) 自动暂停解码，回到前台时自动恢复。
  * **遮挡智能休眠**：当呼出底部菜单、音乐面板、历史记录、书签或系统设置等任何半屏/全屏遮罩层时，后台视频壁纸立刻暂停渲染，完全释放 CPU 和 GPU 算力，保障前台交互动画以 120Hz 极限帧率丝滑呈现；一旦遮罩层关闭，视频背景将即刻恢复播放。

### 4. 🎵 极简音乐播放器与双语独立歌词
* **前台音乐服务 (MusicService)**：基于 Jetpack Media3 ExoPlayer 构建音乐播放服务，以系统 Foreground Service 形式运行，确保在后台播放时不受系统内存清理策略影响。
* **定制化通知栏遥控器**：利用 RemoteViews 深度定制通知栏控制卡片（提供折叠 COLLAPSED 与展开 EXPANDED 两种模式），展示高清唱片背景、歌名、歌手，并支持在通知栏或系统锁屏直接控制播放/暂停、上一首、下一首和停止。
* **双语独立时间轴 LRC 对齐**：支持原歌词与翻译歌词两条独立的时间轴对齐。当音乐播放到特定毫秒时，歌词才同步高亮显示，未达到时间的歌词保持空白。
* **人声提前预读优化**：内置 300ms 歌词预读缓冲算法，完美解决因网络延迟或音频解码造成的人声与歌词显示滞后问题。
* **主页歌词悬浮挂件**：在主页搜索框下方提供实时歌词浮窗。
  * **重组隔离技术**：利用 Compose 延迟状态提供者（Lambda State Provider）设计，歌词在播放滚动时，仅在 `HomeScreenLyrics` 内部进行极小范围的 Text 重绘，主页其他静态组件及搜索框实现 **0 次重复渲染**，极大降低主页耗电量。
* **即时交互反馈**：点击停止/暂停时，主页悬浮歌词与播放器状态瞬间清空，无需等待任何网络延迟或底层播放器回调。
* **播放进度与记忆恢复**：每次退出应用时，系统会自动将当前的播放歌曲、播放列表、以及精确的进度毫秒值持久化保存。下次打开应用时，播放器会自动恢复到上次的歌曲与进度状态（并保持暂停状态，静待用户唤醒），实现无缝断点续播。
* **播放列表滑动删除**：支持在播放列表界面左滑或点击直接删除已缓存的离线歌曲。

### 5. 📥 自研高并发下载管理器
* **OkHttp 并发下载内核**：抛弃了系统不稳定的 DownloadManager，采用 OkHttp 自研高并发下载引擎，支持大文件与音乐的极速下载。
* **实时下载看板**：可视化下载界面，实时计算并显示下载速度（KB/s、MB/s）、当前已下载字节数/总大小、下载进度条百分比，以及下载状态（进行中、暂停、已完成、失败）。
* **下载去重保护**：在开始下载前自动扫描本地文件，如发现同名文件已下载完成，则弹出覆盖/直接打开提示，避免重复消耗流量。
* **自定义存储路径**：支持使用 Android 存储访问框架 (SAF / Document Tree URI) 选择自定义的**音乐下载路径**与**普通文件下载路径**，适配不同手机的存储管理习惯。

### 6. 🔒 安全隐私与深度清理
* **细粒度 Cookies 控制**：提供全局 Cookie 开关；并可展示所有已保存 Cookie 的网站域名，支持针对单个域名进行 Cookie 删除。
* **存储深度清理**：
  * 实时计算并展示应用的 Cache 缓存、临时文件及代码缓存的物理占用大小。
  * 一键清理不仅删除本地缓存文件，还会彻底清空 WebView 的内部浏览缓存、WebStorage 数据库，并一键注销 Cookie 管理器中的全部登录状态，确保个人隐私绝对安全。

### 7. ⚡ UI/UX 细节体验优化
* **触控面积翻倍**：将底部导航栏的按钮点击判定区域扩展至整个单元格（宽度均分，高度填满），彻底解决小屏幕手机容易点不准、点不中导航按钮的问题。
* **无延迟返回交互**：优化了面板关闭时的拦截逻辑。在菜单或音乐面板向下滑动或点击空白处关闭的瞬间，背景遮罩立即放弃触控拦截。用户无需等待面板完全滑下去的 250ms 动画，即可立刻点击底部导航栏进行其他操作，消除动画阻滞感。
* **硬件加速平移动画**：所有滑入/滑出面板均采用 `graphicsLayer { translationY = ... }` 硬件加速位移，不触发组件大小重新计算与重排版，保证低端机型上依然流畅无比。

## 🛠️ 技术栈与底层依赖

* **开发语言**：Kotlin 1.9.x
* **UI 框架**：Jetpack Compose (完全遵循 Material Design 3 规范设计)
* **网络请求**：OkHttp 3 (全局共享单例，复用连接池减少 TCP 握手)
* **数据解析**：Gson (轻量高效的 JSON 编解码)
* **核心组件**：
  * **Android System WebView** (定制化配置，支持 ForceDark、Cookie管理、自定义 UA、下载拦截)
  * **Google Media3 ExoPlayer** (音乐后台播放与主页循环视频背景渲染)
  * **Play Services Location** (fusedLocationClient 实现高效、低功耗的位置信息采集)
  * **SharedPreferences** (轻量级持久化配置存储)

## 📦 如何编译与体积优化

应用中集成了庞大的 `material-icons-extended` 等大体积图标库，通过在构建配置中开启混淆和代码裁剪，APK 的包体积从 **86.7 MB** 极限精简至 **14.3 MB**（精简了近 **82%**）。

### 1. 命令行一键打包 APK
在项目的根目录下打开控制台（如 PowerShell），运行以下命令：
```powershell
# 编译高度混淆并自动使用调试证书签名的发布版 APK
.\gradlew.bat assembleRelease
```
编译完成后，生成的 APK 文件将保存在：
`app/build/outputs/apk/release/app-release.apk`
*(该 APK 已经默认使用调试密钥签名，可以直接发送给他人安装测试)*

### 2. 使用 Android Studio 导出正式版 APK
1. 打开 Android Studio 项目，在顶部菜单选择 **Build** -> **Generate Signed Bundle / APK...**。
2. 选择 **APK**，点击 **Next**。
3. 创建新的或选择已有的签名密钥库 (.jks)，输入密码。
4. 在 Build Variants 中选择 **release** 构建变体。
5. 点击 **Finish**，等待构建完成后，点击右下角提示中的 **locate** 即可找到正式签名发布版 APK。
